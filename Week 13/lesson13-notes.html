An HTML form is a certain section of a webpage where users can input information. In most cases, information entered in a form is sent server-side or to a back-end for processing. In a textbox form, a button is either supplied by input tag (when "submit" chosen as type), or can be customized by you as an image. In the form tag, you can specify a URL to send the inputted information, which is usually a script that dictates how the information will be processed. The most popular server-side script to process inputted information is PHP.

All HTML forms begin with the form tag, which typically has three attributes: name, method, and action. method indicates what format the inputted information should be in, document or URL query string, and action is how you specify the URL to send the information. Within the form tag, you place the input tag to prompt the user for information, although the input tag is not required to be nested within the form tag. The input tag has the following common attribute, name, type, placeholder, required, placeholder, and autofocus. Other available attributes for the input include the size and the maxlength attributes.

The name attribute in the input tag can also be used as a variable name that can be referred to in JavaScript. The allows you to add a caption for a input tag and provide additional context for the web element. The fieldset and the legend tags are used to provide a caption and context for a grouping of form elements, with fieldset defining the group and legend providing the caption and context. The type = hidden attribute for input allows you to send additional form data that isn't publicly visible to the user on the webpage. Besides the text box, there are other form designs you can use to prompt the user to enter data.

One common form design is the checkbox, which allows you to check any number of checkboxes, and the corresponding data associated with the checked boxes can be stored in an array or as a separate variable for each checkbox. The checkbox object has 4 properties: name, value for a checked box, defaultChecked to indicate the default state for a checkbox, and checked is specifies the default state and is a boolean value. Another form design is the radio button, where a user chooses one choice for a list of options. The radio button object, named radio, also has a name, value, defaultChecked, and checked properties, as well as a length property, indicating the number of radio buttons. radio is an array object and each individual radio button and its data is accessed by an index number. A radio button has a click method and a onClick event handler.

Yet another form design that is commonly used is the selection list, which is defined with the select tag, and the options in the list are defined with the option tag. The size attribute for select determines how many items appear before scrolling down, and the multiple attribute allows you to select multiple items. The select object has the following properties: name, length for the number of options and selectedIndex for the index of the currently selected option, or the first selected item for multiple selection. The options object has the following properties: index for the index of the array, defaultSelected for the state of the selected attribute, selected to set the state of the option, name, and text to display text for option. optgroup allows you to group options together in custom categories.

While <input type="text"> can allow you to input one line of text, the textarea tag can allow to input multiple lines of text. textarea has both rows and cols attribute that determines the size of the textbox in lines of text, and a scrollbar will appear if you exceed the amount of text that can fit in the display area. Both the text and the textarea object (as well as the select object) have the blur and the focus methods, which sets and removes where the cursor is respectively. In addition, there is also the select method, which selects all the text in the field, as well as event handlers for focus (onFocus), blur (onBlur), select (onSelect), and for modifying text and moving away from the textbox (onChange).

There are several other type of form designs available with the input type attribute, including email, file, number, range, search, and url. You can use the datalist tag to add a selection of suggested text that the user can click on to input into a textbox. With some of the form designs available in input, data validation is available by default with HTML5, given you don't use the novalidate attribute in form. Another form of data validation is through the pattern in input, which uses regular expressions to only accept input that matches the specified regular expression. It is common to include buttons to submit or clear data with submit and clear for the type attribute in input. You can also use type="image" to make a custom button based on an image, or use type="button" that does nothing by default when you click on it, so you can add an event listener yourself.

The button tag is another way to make buttons to submit information input in a form, and also has the reset, button and the button values available for the type attribute. With the exception of Internet Explorer and Edge, browsers support the output tag where you can describe the output of a form, including any error messages. The form object has itself a submit and a reset method, as well as event handlers for these actions, onSubmit and onReset. With the aforementioned event handlers, you can prevent the data from being submitted or cleared when the appropriate button is clicked.

To access form objects in JavaScript you can use the document object, and the '.' operator to access its derived objects, properties and methods. The brackets "[]" are necessary to access a certain index in array objects, like elements. The getElementById method will be useful to assign form objects to a variable name so you can write cleaner code.
